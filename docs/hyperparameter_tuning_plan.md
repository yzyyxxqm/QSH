# QSH-Net 超参数与诊断计划

> **最后更新：** 2026-04-21

## 当前状态

当前工作重点已经从「常规超参数扫描」切换为「结构诊断优先」。

当前最重要的现实结论不是某个学习率或 batch size 没调好，而是：

1. 旧 M1 的真实训练动力学并不是「Spike + Event + Quaternion 协同」。
2. 旧 M1 更接近 `HyperIMTS + retain 调制 + quaternion 残差`。
3. 现在虽然 `event` 已经不再是死分支，但它仍然不是稳定可靠的主收益来源。
4. `USHCN` 的坏轮更像是活跃主路径在高方差数据上的内部比例失衡，而不是某个单独 gate 没调好。

## 当前成功标准

截至 2026-04-21，当前优化目标已经调整为：

1. 优先优化 `USHCN` 之外的数据集整体表现；
2. `USHCN` 不再以“尽量压低最坏 seed 上界”为首要目标；
3. `USHCN` 更合理的要求是：
   - `itr=5` 中大部分 seed 优于 `HyperIMTS` 论文/项目参考；
   - 允许存在少量坏轮；
   - 但不能为了极少数坏轮而系统性牺牲其他数据集表现；
4. 后续结构改动首先要服务于跨数据集平均表现，而不是只服务于 `USHCN` 的最坏轮。

## 当前不应继续优先做的事

- 继续做大规模学习率、batch size、层数扫描；
- 把问题简单归结为 `event_gate` 没调好；
- 在 `USHCN` 上只看前几轮验证损失就下结论；
- 继续在输出 residual correction 头上叠加各种 gate；
- 继续简单裁剪 `event_proj` 范数；
- 继续统一压低 `innovation_budget`；
- 把问题重新写成常规超参数不足。

## 最近一轮已完成试验与结论

### 1. `event_proj_norm_cap`

已测：

- `epcap5`
- `epcap6`

结论：

- 简单裁剪 `event_proj` 有效范数失败；
- 会误伤好轮，但不能稳定压坏轮；
- 后续不再继续扫相邻阈值。

### 2. `qsh_residual_correction_self_gate`

结果：

- `HumanActivity`: `0.041475 ± 0.000215`
- `USHCN`: `0.190852 ± 0.037656`

结论：

- 对 Human 基本无实质改善；
- 对 USHCN 无法压坏轮；
- 不保留。

### 3. `qsh_residual_correction_confidence_gate`

结果：

- `USHCN`: `0.207944 ± 0.039097`

结论：

- 明显失败；
- 输出端置信门控不是正确方向。

### 4. `innovation_budget_init = 0.75`

结果：

- `USHCN`: `0.194180 ± 0.036793`

结论：

- 统一收紧 event 与 quaternion 共享预算失败；
- 说明不能用整体收缩去解决坏轮。

## 当前工作判断

### 当前更准确的诊断

最近连续失败的结果说明：

1. 问题不在 residual correction 输出头没有加够约束；
2. 问题也不在统一收紧活跃分支预算；
3. `USHCN` 坏轮更像是活跃主路径内部比例失衡；
4. 后续若继续推进，应更靠近主干传播与活跃分支的内部耦合，而不是输出端补丁。

### 当前最该保留的起点

- `qsh_rescorrnorm100_cap003_ushcn_itr5_local`

理由：

- 它仍然拥有 4 个非常强的好轮；
- 当前还没有单因素改动能在保住好轮的同时压掉坏轮；
- 所以后续若继续试，应默认以它为直接起点。

### 当前最重要的优先级

1. 优先优化 `HumanActivity / P12 / MIMIC_III / MIMIC_IV` 等非 `USHCN` 数据集；
2. `USHCN` 改为约束项，而不是唯一主目标；
3. 每次只改一个核心因素；
4. `USHCN` 至少看 `itr>=5`，但重点看“大部分 seed 是否优于 `HyperIMTS` 论文/项目参考”；
5. 不再为了压极少数坏轮而默认采用更强保守化改动。

## 后续建议方向

如果继续推进，当前更值得尝试的是：

1. 更贴近有效活跃分支表达增强、但不过度伤害 `USHCN` 主体 seed 的结构改动；
2. 更贴近 route / propagation 选择机制的温和重参数化；
3. 更贴近 quaternion 有效残差占比的按样本控制；
4. 活跃主路径之间的内部比例修正；

而不是：

- 输出端继续叠 gate；
- 再做统一预算收缩；
- 回到常规超参数扫描。
